Git的使用

2022/10/31 git git

Git 是一个免费的、开源的分布式版本管理系统。GIt 是Linus Torvalds 为了更好的帮助管理Linux内核开发而开发的版本控制软件。

git官网：[GIT官网](Git (git-scm.com))

1.Git常用命令

以下的操作全在git bash中完成。

命令 作用
git config –global user.name 用户名 设置用户签名
git config –global user.email 邮箱 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件名 添加到暂存区
git commit -m “日志信息” 文件名 提交到本地库
git reflog 查看历史记录
git reset –hard 版本号 版本穿梭

1.1 设置用户签名

git设置用户签名的命令是：

git  config --global user.name 用户名
git config  --global user.email 邮箱

当首次安装号git后需要设置用户签名，用户签名设置后才可以进行提交操作，否则无法进行提交，用户签名可以在提交代码是用于确认是哪个用户（你电脑上的本地的git客户端）执行的此次操作。

设置好用户签名后可以在本地用户的目录下找到.gitconfig文件，打开可以看到刚刚设置的签名

1.2 初始化本地库

要想使用git来管理版本必须先要初始化本地库。git初始化本地库的命令是：

git init

初始化后可以在git管理的文件加下看到.git文件夹

1.3 查看本地库状态

查看本地库状态的命令:

git status

在git管理的目录下新建hello.txt文件，并写入一些数据，使用·git status命令查看如下：

通过如上命令可以知道，现在git在master主分支上，还没有进行commit，所以git无法追踪hello.txt文件，也无法对hello.txt文件进行版本管理。

1.4 添加文件到暂存区

添加文件到暂存区的命令：

git add 文件名

将上一步的文件添加到暂存区后再次查看本地库状态如下：

可以看到文件颜色变为了绿色，说明现在该文件可以被追踪到，并可以进行提交操作了。

1.5 删除暂存区的文件

删除暂存区的文件的命令是：

git  rm --cached 文件名

从暂存区移除文件后可以看到文件又变回了之前无法追踪的情况了。

1.6 提交文件到本地库

提交文件到本地库的命令：

git commit -m "提交信息" 文件名

文件被提交到本地库后就会形成该库的历史版本信息。

1.7 查看历史记录

git reflog                    #查看缩略的历史记录
git log                       #查看详细的历史记录

如上图所示：git reflog命令是查看缩略的历史记录，git log命令是产看详细的历史记录。

最前面的那一串是版本号，最后面的是我们提交时所描述的该次提交的信息，查看详细的历史记录可以看到提交者的用户签名。

2.Git的使用

2.1 版本穿梭

版本穿梭的命令：

git reset --hard 版本号

git的版本穿梭可以让我们切换到任意的历史版本。

使用vim hello.txt命令修改hello.txt文件，在其后添加“first update hello.txt”后，再次使用git status命令查看git状态信息。可以看到git提示我们文件已经修改（modified）了。且文件名是红色的，红色代表该文件还没有添加到暂存区。

使用git add 命令将修改后的文件添加到暂存区后，再次查看git信息。可以看到文件已经被放入了暂存区。

然后我们提交该文件到git本地库，使用git reflog查看git 的历史记录。

我们可以看到此时，历史记录中有了两份记录，一份是我们第一次进行提交时的，另一个是我们修改文件后进行提交的。且此时HEAD指向了第二次提交的版本。我们使用cat 命令查看hello.txt文件的内容。

现在是第二个版本，如果我们想回到第一个版本，可以使用git reset --hard 版本号命令。

使用cat 命令再次查看hello.txt文件的内容。我们发现该文件变回了第一个版本。

再将其变回第二个版本。

注意：版本穿梭会影响工作区，需要谨慎使用。

git的版本穿梭底层是使用指针，head指针指向当前的分支，当前的分支指针指向各个版本。

2.2 Git分支（重要）

命令 作用
git branch 分支名 创建分支
git branch -v 查看分支
git checkout 分支名 切换分支
git merge 分支名 把指定的分支合并到当前分支上
git branch -m 旧分支名 新分支名 重新命名分支
  • 查看分支：git branch -v：可以查看当前已经创建的分支
  • 创建分支：git branch 分支名：创建一个新的分支
  • 切换分支：git checkout 分支名：从当前分支切换到另一个分支
  • 分支改名：git branch -m 旧分支名 新分支名：重命名分支
  • 合并分支：git merge 分支名：把指定的分支合并到当前分支上。

分支合并冲突：合并分支时，如果两个分支在同一个文件的同一个位置有两套完全不同的修改。git无法替我们决定要使用哪一个。此时必须人为决定选择哪一个作为新的内容。

3.Github操作

github官网：https://github.com/

3.1github创建远程仓库

点击new repository新建一个远程代码仓库

3.2创建远程仓库别名

git remote -v  						#查看当前所有远程仓库的地址别名
git remote add 别名  地址            #给地址取一个别名

给刚刚我们创建好的远程仓库取别名并查看。

3.3推送本地仓库到远程仓库

git push 别名 分支名

在推送时，最好确保你推送的代码是稳定、无bug版的。

推送完成后，就可以在远程仓库上看到我们刚刚推送的代码了。

3.4拉取远程仓库到本地库

git pull 别名 分支名

3.5克隆远程仓库代码

git clone 远程地址

3.6配置ssh免密登录

生成ssh密钥的命令：

ssh-keygen -t rsa -C "你的github邮箱"

输入命令后按3下回车键。

生成密钥后可以在看到.ssh文件夹

复制公钥到github账号setting下的SSH and GPG keys下SSH，新建SSH将刚刚复制的公钥粘贴上去。此时ssh免密登录就配置好了。

4.IDEA中使用git

4.1 配置git的忽略文件

eclipse和IDEA中都有一些软件的配置文件，提交git时需要忽略这些文件，这些文件与我们所写的项目代码无关。将它们忽略可以屏蔽掉IDE工具之间的差异。

创建忽略文件xxxx.ignore(建议取名为git.ignore)，将该文件放在系统用户目录下。输入如下内容。

# compiled class file
*.class

#log file 
*.log

#blueJ files
*.ctxt

.mtj.tmp/

#packageFiles
*.jar
*.war
*.ear
*.zip
*.tar.gz
*.rar


hs_err_pid*

.classpath
.project
.settings
target
.idea
*.iml

然后在.gitconfig文件中配置git.ignore文件位置。

[core]
    excludesfile = C:/Users/luo/git.ignore

4.2 IDEA使用git

4.2,1 IDEA定位git

打开IDEA，然后点击File下的settings中的version control下的git。

找到git.exe所在的位置。此时，我们可以用IDEA来使用git了。

4.2.2 IDEA中git的操作
  • 初始化本地库：点击上方菜单VCS点击create git repository。
  • 添加到暂存区：右击文件选择：git –> add
  • 提交到本地库：右击文件选择：git –> commit directory 。提交后可以看到暂存区中的文件
  • 切换代码版本：坐下角git点击某一版本点击checkout revision
  • 创建分支：右下角new branch
  • 切换分支：右下角选择分支名,来切换分支
  • 合并分支：右下角merge … inot …

4.3 IDEA使用Github

找到settings中的github(如果没有则使用点击settings中的plugins安装github插件)。

4.3.1 设置github账号

使用token口令登录github账号。token口令生成如下：

  1. 点击github账号的settings
  2. 往下翻找，找到Developer Settings
  1. 找到Personal access tokens下的tokens点击generate new token生成token口令。

注意：token口令生成一次后就不可以在查看了，只能重新生成新的token。

生成token后，利用token在IDEA设置gihub账号。

  • IDEA推送项目到github：右击Git –> github –> share project on github
  • push到github：右击Git –> push
  • pull代码：右击Git –> pull

克隆项目到本地：点击get from vcs，填入克隆项目的URL
