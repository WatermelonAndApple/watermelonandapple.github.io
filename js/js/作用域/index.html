Js变量的作用域

2023/3/1 js js

1.局部作用域

局部作用域分为函数作用域和块作用域。

1.1函数作用域

在函数内部声明的作用域，只能在函数内部被访问，外部无法直接访问。

   function test(){
            const a = 10;
            console.log(a,"函数内部访问");
        }
        test();
        console.log(a,"函数外部访问");

打开浏览器控制台，可以看到控制台打印了“函数内部访问”，而没有打印””函数外部访问”,而且显示”a is not defined”。

  • 函数内部声明的变量，在函数外部无法直接访问。
  • 函数的参数也是函数的内部定义的局部变，在函数外部无法直接访问。
  • 不同函数内部声明的变量无法互相访问。

1.2块级作用域

在JavaScript中使用{ }包裹的代码称为代码块，代码块内部声明的变量外部有可能无法被访问。

 {
            let a = 10;
            console.log(a,"块级作用域内");
 }
        console.log(a,"块级作用域外");

注意:使用var在块级作用域中定义的变量，外部可以访问。

  {
            var a = 10;
            console.log(a,"块级作用域内");
}
        console.log(a,"块级作用域外");
  • 使用let声明的变量以及使用const声明的常量都会产生块级作用域，var声明的变量不会产生块级作用域。
  • 不同代码块之间的变量无法相互访问。

2.全局作用域

在script标签和.js文件的最外层就是就是全局作用域，在次声明的变量在函数内部也可以被访问。全局作用域中声明的变量，任何其它作用域都可以被访问。

    //a,b是全局作用域
        let a = 1;
        let b =2;
        function f(){
             
        }
        f();

3.作用域链

作用域链的本质是底层的变量查找机制。

  • 在函数执行时，会优先在当前函数作用域中查找变量。
  • 如果当前作用域查找不到则会依次逐级查找父级作用域直到全局作用域。
let a = 1;
function f(){
    let a =10;
    function g(){
        a =20;
        console.log(a);
    }
    console.log(a);
    g();
}
f();
console.log(a);

执行以上的代码，可以看到，依次打印了10、20、1。

总结：

  1. 嵌套关系的作用域串联起来形成了作用域链。
  2. 相同作用域链中按照从小到大的规则查找变量。
  3. 子作用域能够访问父作用域，父作用域无法访问子作用域。
