1.MySQL简介

  • 历史：
    • 有瑞典的MySQL AB公司创立，1995年开发出MySQL。
    • 2008年，MySQL被SUN公司收购。
    • 2009年，Oracle公司受够了SUN公司，进而Oracle公司获取了MySQL。
    • 2016年，MySQL8.0.0版本推出。
  • 特点:
    • 开源的、关系型的数据库。
    • 支持千万级别的数据量的存储。

2.SQL概述

2.1 概述

SQL(Structured Query Language, 结构化查询语言)是使用关系模型和数据库应用语言，其直接与数据库打交道，有IBM上世纪70年代开发出来。后来由美国国家标准局（ANSI）开始着手制定SQL标准，先后有SQL-86,SQL-89,SQL-92,SQL-99等标准。

SQL分类

SQL在语言功能上主要分为以下3大类：

  • DDL(Data Definition Languages、数据定义语言)，这些语句定了不同的数据库、表、视图、索引等数据库对象，还可以用来创建、删除、修改数据库和数据表的结构。
  • DML（Data Manipulation Language、数据操作语言），用于添加、删除、更新和查询数据库记录，并检查数据完整性。
  • DCL（Data Control Language、数据控制语言），用于定义数据、表、字段、用户的权限和安全级别。
/*
1.SQL的分类
DDL:数据定义语言
如：
CREATE \ ALTER \ DROP \ RENAME \ TRUNCATE
DML:数据操作语言
如：
INSERT \ DELETE \ UPDATE \ SELECT 
DCL:数据控制语言
如：
COMMIT \ ROLLBACK \ SAVEPOINT \ GRANT \ REVOKE
*/

2.2 SQL语言的规则与规范

MySQL在Windows环境下的大小写并不敏感。

MySQL在Linux环境下的大小写是敏感的。

推荐采用如下的书写规范：

  • 数据库名、表名、字段名、字段别名、表别名等小写。
  • SQL关键字、函数名、绑定变量等都大写。
  • 基本规则
    • SQL语句可以写在一行或多行。为了提高可读性，各句分行写，必要时缩进
    • 每条命令以；或\g或\G结束
    • 关键字不能缩写也不能分行
    • 字符串和日期时间类型的数据可以使用单引号（’’）表示
    • 列的别名，尽量使用双引号（””），而且不建议省略as
  • MySQL注释

    • 单行注释:#注释以及--注释。

    • #这是单行注释
-- 这也是单行注释

    • 多行注释：/*注释*/

    • /*这是一条
多行注释*/
